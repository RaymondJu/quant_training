# ML 回测窗口与结果可信度核查

本文档用于记录当前项目中机器学习模型回测窗口、调参流程和结果可信度的核查结论。

## 1. 核查结论摘要

当前没有发现主 ML 默认模型存在“为了让结果变好而调整滑动窗口”的问题。

主 ML 默认模型仍使用固定 walk-forward 结构：

- 训练窗口：24 个月
- 验证窗口：3 个月
- 预测窗口：逐月预测
- 第一个预测月：2017-10
- ML 回测区间：2017-10 至 2025-11
- 有效回测月数：98 个月

但是，当前主汇总表存在一个展示口径问题：

- 传统 baseline 使用 125 个月，2015-07 至 2025-11
- ML 模型使用 98 个月，2017-10 至 2025-11

因此，原始总表不能直接作为 baseline 与 ML 的完全公平横向比较。正式比较时，应将 baseline 同样截取到 2017-10 至 2025-11。

此外，后续调参结果，尤其是 `XGBoost-local` 和 `CatBoost-local`，属于二阶段调参开发结果，不能当作完全无偏的独立样本外回测。

## 2. 主 ML walk-forward 设置

代码中主 ML 的窗口设置为：

- `LGBM_TRAIN_WINDOW = 24`
- `VAL_WINDOW = 3`
- `start_idx = train_window + val_window`

第一个 fold 的实际时间切分为：

- 训练集：2015-07 至 2017-06
- 验证集：2017-07 至 2017-09
- 预测月：2017-10

这说明 ML 的起点晚于 baseline，是因为需要先积累训练集和验证集，而不是人为挑选了一个更容易的回测起点。

## 3. 对三种风险情况的判断

### 情况 A：最严重情况

描述：

原来 baseline 和 ML 都是同一套窗口，比如 24m train / 3m val / 3m test，后来为了让 ML 表现更好，把 ML 改成 36m train / 6m val 等其他窗口，导致 ML 回测起点推后、表现变好。

判断：

当前没有发现这种情况。

主 ML 默认模型仍然是 24 个月训练窗口加 3 个月验证窗口。没有发现把主 ML 默认模型改成长窗口以提升结果的证据。

### 情况 B：中等风险情况

描述：

窗口本身没有动，但起始时间、预热期或股票池筛选条件发生变化，导致 ML 跑在了更容易的时期。

判断：

主 ML 的较晚起点来自训练和验证预热期，不是单独挑选时期。

但是，当前主汇总表中 baseline 和 ML 的统计区间不一致，会让读者误以为 ML 的提升幅度比真实同窗口比较更大。因此展示时需要统一口径。

### 情况 C：可接受但需要修正展示

描述：

只是 baseline 回测区间和 ML 回测区间不一致，baseline 没有截短到 ML 的共同窗口。

判断：

当前确实存在这种情况。

主表中 baseline 是 125 个月，ML 是 98 个月。因此正式比较时，需要补充一张 2017-10 至 2025-11 的统一窗口横比表。

## 4. 当前主汇总表的问题

当前 `output/analysis/performance_table.csv` 中的区间如下：

| 策略类型 | 回测区间 | 月数 |
|---|---:|---:|
| Equal-weight / IC-weight / ICIR-weight | 2015-07 至 2025-11 | 125 |
| LightGBM / CatBoost / XGBoost / RandomForest / Ridge | 2017-10 至 2025-11 | 98 |

这张表可以展示各策略自身表现，但不能直接用于严格判断“ML 是否显著优于 baseline”。

原因是两个区间的市场环境不同。baseline 多包含了 2015-07 至 2017-09 这段时期，而 ML 不包含。

## 5. 统一 98 个月窗口后的结果

将 baseline 也截取到 ML 的共同窗口 2017-10 至 2025-11 后，结果如下：

| 策略 | 年化收益 | Sharpe | 最大回撤 | 月胜率 | 月数 |
|---|---:|---:|---:|---:|---:|
| Equal-weight | 18.98% | 0.928 | -27.56% | 59.18% | 98 |
| IC-weight | 19.00% | 0.948 | -24.33% | 58.16% | 98 |
| ICIR-weight | 19.27% | 0.953 | -23.81% | 58.16% | 98 |
| LightGBM | 18.50% | 0.880 | -22.86% | 59.18% | 98 |
| CatBoost | 20.13% | 0.914 | -24.76% | 58.16% | 98 |
| XGBoost | 21.09% | 1.000 | -22.47% | 59.18% | 98 |
| RandomForest | 21.50% | 1.009 | -22.87% | 59.18% | 98 |
| Ridge | 20.20% | 1.018 | -20.85% | 62.24% | 98 |

统一窗口后的结论更温和，也更可信：

- `LightGBM` 没有跑赢同期 baseline。
- `CatBoost` 年化收益高于 baseline，但 Sharpe 没有明显超过。
- `XGBoost`、`RandomForest`、`Ridge` 相对同期 baseline 有增量提升。
- `RandomForest` 默认模型年化最高。
- `Ridge` 默认模型 Sharpe 最高。

因此，不能说 ML “全面碾压”传统因子组合。更准确的结论是：部分 ML 模型在统一窗口下能够提供温和但可见的增量。

## 6. 调参结果的可信度

### 第一轮 Optuna 调参

第一轮调参使用较早期样本选择参数：

- 调参截止：2019-09
- 方法：Optuna / TPE
- 然后用选中参数跑完整 walk-forward 回测

第一轮调参结果：

| 策略 | 年化收益 | Sharpe | 最大回撤 |
|---|---:|---:|---:|
| LightGBM-tuned | 18.90% | 0.915 | -21.57% |
| CatBoost-tuned | 19.84% | 0.944 | -20.57% |
| XGBoost-tuned | 19.60% | 0.942 | -23.24% |
| RandomForest-tuned | 22.03% | 1.026 | -22.84% |
| Ridge-tuned | 20.88% | 1.036 | -18.73% |

这类结果可以作为模型开发结果使用，但不能被解释成完全独立、完全无偏的样本外检验。

相对较稳健的是：

- `Ridge-tuned`
- `RandomForest-tuned`

因为它们相对默认版本确实有所提升，且不是通过改变主 ML 回测窗口实现。

### 第二阶段局部精调

第二阶段局部精调只针对：

- `XGBoost`
- `CatBoost`

这一阶段是在看到第一轮 `XGBoost-tuned` 和 `CatBoost-tuned` 表现不理想后进行的，并将调参窗口延长到 2020-09。

第二阶段结果：

| 策略 | 年化收益 | Sharpe | 最大回撤 |
|---|---:|---:|---:|
| CatBoost-local | 21.03% | 0.985 | -23.28% |
| XGBoost-local | 21.56% | 1.008 | -22.32% |

这两个结果说明，上一次 `XGBoost` 和 `CatBoost` 调参变差，很可能与搜索范围和调参窗口设置有关。

但是，这两个结果包含更强的研究者自由度，应标注为“二阶段局部调参开发结果”，不应作为完全无偏的最终回测结果。

## 7. 当前推荐的结果使用方式

正式展示时，建议分成三类表述。

### 7.1 全历史 baseline 表

用途：

说明传统因子组合本身是否有效。

推荐表述：

在 2015-07 至 2025-11 的全历史区间内，传统多因子组合表现有效，其中 `IC-weight` 年化收益约 18.25%，Sharpe 约 0.833。

注意：

这张表不能直接和 ML 表做严格横比，因为 ML 从 2017-10 开始。

### 7.2 统一窗口横比表

用途：

严格比较 baseline 和 ML。

推荐表述：

由于 ML 需要 24 个月训练窗口和 3 个月验证窗口，因此 ML 回测从 2017-10 开始。将 baseline 同样截取到 2017-10 至 2025-11 后，`XGBoost`、`RandomForest`、`Ridge` 相对传统 baseline 有一定提升，但提升幅度是温和的。

这是最适合作为主结论的表。

### 7.3 调参开发表

用途：

展示模型开发和优化过程。

推荐表述：

经过 Optuna 调参后，`Ridge-tuned` 和 `RandomForest-tuned` 表现进一步提升。`XGBoost-local` 和 `CatBoost-local` 是二阶段局部精调结果，可以作为模型开发参考，但不应等同于完全无偏回测。

## 8. 最终可信结论

当前项目中，主 ML 默认模型没有发现通过调整滑动窗口进行 backtest overfitting 的证据。

但是，原主汇总表确实混用了不同回测区间：

- baseline：125 个月
- ML：98 个月

因此，正式比较必须使用统一的 98 个月窗口。

在统一窗口下，结论应改为：

- 传统 baseline 本身已经很强，Sharpe 接近 0.95。
- `LightGBM` 没有显著优于 baseline。
- `CatBoost` 略有改善，但优势不强。
- `XGBoost`、`RandomForest`、`Ridge` 是更有说服力的 ML 增强模型。
- 调参后的 `Ridge-tuned` 和 `RandomForest-tuned` 可以作为模型开发后的增强结果。
- `XGBoost-local` 和 `CatBoost-local` 应作为二阶段调参结果谨慎展示。

最稳妥的一句话总结：

> 主 ML 默认结果没有因为滑动窗口调参而作废；但当前总表需要统一回测区间后再比较。调参和二阶段局部精调结果应单独标注为模型开发结果，不能和默认模型同等看待。
